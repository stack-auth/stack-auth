This is a detailed reference for email-related types in Stack Auth. If you're looking for a more high-level overview, please refer to our [guide on the email system](/docs/concepts/emails).

# `SendEmailOptions`

Options for sending emails via the `sendEmail` method on `StackServerApp`.

## Table of Contents

```typescript
type SendEmailOptions = {
  userIds: string[];
  themeId?: string | null | false;
  subject?: string;
  notificationCategoryName?: string;
  html?: string;
  templateId?: string;
  variables?: Record<string, any>;
};
```

## Properties

<ResponseField name="userIds" type="string[]" required>
  An array of user IDs that will receive the email. All users must exist in your
  Stack Auth project.
</ResponseField>

<ResponseField name="subject" type="string">
  Optional email subject line. If you use a template, this overrides the
  template's default subject.
</ResponseField>

<ResponseField name="html" type="string">
  Custom HTML content for the email. This cannot be used together with
  `templateId` or `variables`.
</ResponseField>

<ResponseField name="templateId" type="string">
  ID of the email template to use. This cannot be used together with `html`.
</ResponseField>

<ResponseField name="variables" type="Record<string, any>">
  Optional variables to substitute in the template. Only used when `templateId`
  is provided.
</ResponseField>

<ResponseField name="themeId" type="string | null | false">
  A string ID for a custom theme, `null` for no theme, or `false` to use the
  default theme.
</ResponseField>

<ResponseField name="notificationCategoryName" type="string">
  Optional notification category name for user preferences. Users can opt in or
  out of specific categories through their account settings.
</ResponseField>

## Usage

Developers choose between two approaches:

<Tabs>
  <Tab title="Custom HTML">
    Provide custom HTML directly:

    ```typescript
    await stackServerApp.sendEmail({
      userIds: ["user-id-1"],
      subject: "Hello!",
      html: "<h1>Welcome</h1><p>Thanks for signing up!</p>",
    });
    ```

  </Tab>
  <Tab title="Template">
    Use a template with variable substitution:

    ```typescript
    await stackServerApp.sendEmail({
      userIds: ["user-id-1"],
      templateId: "welcome-template",
      variables: {
        userName: "John Doe",
        activationUrl: "https://app.com/activate/token123",
      },
    });
    ```

  </Tab>
</Tabs>

<Note>
  The `notificationCategoryName` and `themeId` properties work independently
  with either approach.
</Note>
