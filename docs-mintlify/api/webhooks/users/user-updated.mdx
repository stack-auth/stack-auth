## Event Type

`user.updated`

## Description

This event is triggered when a user is updated.

## Payload

<ResponseField name="type" type="string">
  The event type. Always `"user.updated"`.
</ResponseField>

<ResponseField name="data" type="object">
  The user object after the update.
  <Expandable>
    <ResponseField name="id" type="string">
      The unique identifier for the user.
    </ResponseField>
    <ResponseField name="primary_email" type="string | null">
      The primary email address of the user, or `null` if not set.
    </ResponseField>
    <ResponseField name="display_name" type="string | null">
      The display name of the user, or `null` if not set.
    </ResponseField>
    <ResponseField name="signed_up_at_millis" type="integer">
      The timestamp when the user signed up, in milliseconds since epoch.
    </ResponseField>
  </Expandable>
</ResponseField>

<CodeGroup>
```json Example Payload
{
  "type": "user.updated",
  "data": {
    "id": "user-id-123",
    "primary_email": "user@example.com",
    "display_name": "Updated Name",
    "signed_up_at_millis": 1700000000000
  }
}
```
</CodeGroup>
