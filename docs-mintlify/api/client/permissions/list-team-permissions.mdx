---
title: "List Team Permissions"
description: "List team permissions of the current user. user_id=me must be set for client requests."
api: "GET /api/v1/permissions/team-permissions"
---

The combination of `(team_id, user_id, permission_id)` serves as a unique identifier for each permission record. Note that permissions with identical IDs may exist across multiple teams.

## Request

### Query Parameters

<ParamField query="team_id" type="string">
  Filter with the team ID. If set, only the permissions of the members in a specific team will be returned.
</ParamField>

<ParamField query="user_id" type="string">
  Filter with the user ID. If set, only the permissions this user has will be returned. Client request must set `user_id=me`.
</ParamField>

<ParamField query="permission_id" type="string">
  Filter with the permission ID. If set, only the permissions with this specific ID will be returned.
</ParamField>

<ParamField query="recursive" type="boolean">
  Whether to list permissions recursively. If set to `false`, only the permissions the users directly have will be listed. If set to `true`, all the direct and indirect permissions will be listed.
</ParamField>

## Response

<ResponseField name="200" type="object">
  Successful response with the list of team permissions.
</ResponseField>
