---
title: "Update Notification Preference"
description: "Enable or disable a specific notification category for a user."
api: "PATCH /api/v1/emails/notification-preference/{user_id}/{notification_category_id}"
---

## Request

<ParamField header="x-stack-publishable-client-key" type="string" required>
  The publishable client key for authentication.
</ParamField>

<ParamField header="x-stack-access-token" type="string" required>
  The access token for the authenticated user.
</ParamField>

<ParamField header="x-stack-project-id" type="string" required>
  The project ID.
</ParamField>

<ParamField path="user_id" type="string" required>
  The user identifier, or `me` to reference the currently authenticated user.
</ParamField>

<ParamField path="notification_category_id" type="string" required>
  The identifier for the notification category to modify.
</ParamField>

<ParamField body="enabled" type="boolean" required>
  Whether to enable or disable the notification category for this user.
</ParamField>

## Response

<ResponseField name="notification_category_id" type="string">
  The identifier of the notification category.
</ResponseField>

<ResponseField name="enabled" type="boolean">
  Whether the notification category is now enabled for the user.
</ResponseField>

<ResponseField name="category_name" type="string">
  The display name of the notification category.
</ResponseField>

<CodeGroup>
```bash cURL
curl -X PATCH 'https://api.stack-auth.com/api/v1/emails/notification-preference/me/marketing' \
  -H 'Content-Type: application/json' \
  -H 'x-stack-publishable-client-key: YOUR_PUBLISHABLE_CLIENT_KEY' \
  -H 'x-stack-access-token: YOUR_ACCESS_TOKEN' \
  -H 'x-stack-project-id: YOUR_PROJECT_ID' \
  -d '{
    "enabled": false
  }'
```

```json 200 Response
{
  "notification_category_id": "marketing",
  "enabled": false,
  "category_name": "Marketing Emails"
}
```
</CodeGroup>
