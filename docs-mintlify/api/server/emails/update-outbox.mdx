## Description

Modify emails within the outbox system. Users can revise message content, toggle sending states, or remove emails entirely. The operation only succeeds for emails in specific states: `paused`, `preparing`, `rendering`, `render-error`, `scheduled`, `queued`, or `server-error`.

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the outbox email entry.
</ParamField>

## Query Parameters

<ParamField query="status" type="string">
  Status parameter.
</ParamField>

<ParamField query="simple_status" type="string">
  Simplified status parameter.
</ParamField>

## Request Body

A request body is required for this operation.

## Response

**Status Code:** `200` - Successful response
