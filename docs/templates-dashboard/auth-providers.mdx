# Authentication Providers

Configure OAuth providers and social login options to give your users multiple ways to authenticate with your application.

## Overview

Stack Auth supports a wide range of authentication providers:
- Social providers (Google, GitHub, Facebook, etc.)
- Enterprise providers (Microsoft, SAML, etc.)
- Custom OAuth providers
- Passwordless options

## Supported Providers

### Popular Social Providers
- **Google**: OAuth 2.0 integration with Google accounts
- **GitHub**: Perfect for developer-focused applications
- **Facebook**: Social login for consumer applications
- **Discord**: Great for gaming and community apps
- **Twitter/X**: Social media integration

### Enterprise Providers
- **Microsoft**: Azure AD and Office 365 integration
- **SAML**: Enterprise single sign-on
- **LinkedIn**: Professional network integration

### Configuration Steps

1. **Create OAuth Application**: Set up your app with the provider
2. **Configure Credentials**: Add client ID and secret to Stack Auth
3. **Set Callback URLs**: Configure redirect URLs for your application
4. **Test Integration**: Verify the authentication flow works correctly

## Provider Settings

### OAuth Configuration
Each provider requires:
- Client ID
- Client Secret
- Authorized redirect URIs
- Scope permissions

### Advanced Options
- Custom button styling
- Provider-specific settings
- User data mapping
- Permission scopes
