Stack Auth supports CLI authentication for terminal applications. The flow opens a browser for sign-in, polls for completion, and returns a refresh token your CLI can use for authenticated requests.

## JavaScript/TypeScript SDK

Use the `promptCliLogin` method from `StackClientApp` to authenticate users in Node.js CLI applications.

```ts
import { StackClientApp } from "@stackframe/stack";

const app = new StackClientApp({
  projectId: "your-project-id",
  publishableClientKey: "your-publishable-client-key",
  tokenStore: "memory",
});

const refreshToken = await app.promptCliLogin({
  appUrl: "https://your-app-url.example.com",
});

console.log("User authenticated! Refresh token:", refreshToken);
```

### Options

| Option | Type | Description |
|--------|------|-------------|
| `appUrl` | `string` | Base URL of your app (for the login page) |
| `expiresInMillis` | `number?` | How long the login attempt is valid |
| `maxAttempts` | `number?` | Max polling attempts (default: Infinity) |
| `waitTimeMillis` | `number?` | Time between poll attempts (default: 2000ms) |
| `promptLink` | `function?` | Callback to display login URL and verification code |
| `anonRefreshToken` | `string?` | Anonymous refresh token to attach to this login |

### Custom Login Display

Use the `promptLink` callback to customize how the login URL and verification code are displayed to users:

```ts
const refreshToken = await app.promptCliLogin({
  appUrl: "https://your-app-url.example.com",
  promptLink: (url, loginCode) => {
    console.log("\n🔐 To authenticate, visit:");
    console.log(`   ${url}`);
    console.log(`\n   Verification code: ${loginCode}\n`);
  },
});
```

### Anonymous Session Attachment

If your CLI uses anonymous users for local state before sign-in, you can attach that anonymous session to the login. When the user signs in, the CLI receives tokens for the authenticated user while preserving the anonymous session data.

```ts
// Get the anonymous user's refresh token
const anonRefreshToken = await app.getRefreshToken();

// Start CLI login with the anonymous session attached
const refreshToken = await app.promptCliLogin({
  appUrl: "https://your-app-url.example.com",
  anonRefreshToken: anonRefreshToken,
});
```

<Info>
Anonymous account data is not merged into the signed-in user. The anonymous user remains unchanged, and the CLI receives tokens for the authenticated user's session.
</Info>

## Python Template

For Python CLI applications, we provide a template that handles the authentication flow. [Download it here](https://github.com/stack-auth/stack-auth/tree/main/docs/public/stack-auth-cli-template.py) and copy it into your project:

```
└─ my-python-app
   ├─ main.py
   └─ stack_auth_cli_template.py
```

Then import the `prompt_cli_login` function:

```py
from stack_auth_cli_template import prompt_cli_login

# Prompt the user to log in
refresh_token = prompt_cli_login(
  app_url="https://your-app-url.example.com",
  project_id="your-project-id-here",
  publishable_client_key="your-publishable-client-key-here",
)

if refresh_token is None:
  print("User cancelled the login process. Exiting")
  exit(1)

# Store the refresh token for persistent sessions
# You can now use the REST API with the refresh token
```

### Using the Refresh Token

After authentication, use the refresh token to get access tokens for API requests:

```py
def stack_auth_request(method, endpoint, **kwargs):
  # See Stack Auth's Getting Started section for implementation
  # https://docs.stack-auth.com/python/getting-started/setup
  pass

def get_access_token(refresh_token):
  access_token_response = stack_auth_request(
    'post',
    '/api/v1/auth/sessions/current/refresh',
    headers={
      'x-stack-refresh-token': refresh_token,
    }
  )
  return access_token_response['access_token']

def get_user_object(access_token):
  return stack_auth_request(
    'get',
    '/api/v1/users/me',
    headers={
      'x-stack-access-token': access_token,
    }
  )

user = get_user_object(get_access_token(refresh_token))
print("The user is logged in as", user['display_name'] or user['primary_email'])
```
